# MLB HR Props Pipeline

Daily dashboard ranking MLB batters by HR probability via a pitch-type collision model.

## Setup

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Add your OpenWeatherMap API key (free tier at openweathermap.org)
set OPENWEATHER_API_KEY=your_key_here   # Windows
export OPENWEATHER_API_KEY=your_key_here  # Mac/Linux

# 3. Run the pipeline (generates data/data.json)
python pipeline.py

# 4. Deploy to GitHub Pages
python deploy.py

# Or deploy without re-running the pipeline
python deploy.py --skip-pipeline

# Or build locally without pushing
python deploy.py --dry-run
```

## Daily Workflow

```
8 AM  — python pipeline.py        # pulls slate + probable pitchers
Noon  — python pipeline.py        # re-run after lineups confirmed
       python deploy.py --skip-pipeline   # push updated data
```

## Without pybaseball

The pipeline runs in **seed mode** if pybaseball isn't installed — it uses
hardcoded 2025 stats for known players and shows the dashboard structure.
Install pybaseball to enable live Statcast pulls.

## Scoring Model

| Factor | Weight | Source |
|--------|--------|--------|
| EV + Barrel rate | 25% | pybaseball Statcast |
| Pitcher HR allowed by pitch type | 20% | pybaseball |
| Batter SLG/run factor vs pitch type | 20% | pybaseball |
| Park HR factor | 15% | Hardcoded from Statcast |
| Platoon split (L/R) | 10% | MLB Stats API |
| Weather multiplier | 5% | OpenWeatherMap |
| Recent form L14D | 5% | pybaseball |

## Tuning

- Edit `WEIGHTS` dict in `pipeline.py` to adjust factor weights
- Edit `PARK_FACTORS` to update park HR indexes each season
- Edit `STADIUM_CF_BEARING` to update wind direction mappings
- Edit `ELITE_FADE` to update the auto-fade pitcher list

## Data Blending (2025 vs 2026)

- < 30 PA in 2026: use 2025 data only
- 30–200 PA in 2026: linear blend (30 PA = 15% 2026, 200 PA = 100% 2026)
- > 200 PA in 2026: use 2026 data only
